# PinPig 화면 설계

---

## IA (Information Architecture)

### 전체 구조

```
PinPig
│
├── 온보딩 (최초 1회)
│   ├── 웰컴
│   └── 예산 설정
│
├── 메인 탭 (Bottom Navigation)
│   ├── 홈 (/)              ← 기본 탭
│   ├── 내역 (/history)
│   ├── 통계 (/stats)
│   └── 설정 (/settings)
│
└── 모달/오버레이
    ├── 거래 입력 (FAB)
    ├── 거래 상세/수정
    └── 카테고리 선택
```

### 네비게이션 맵

```
┌─────────────────────────────────────────────────────────┐
│                                                         │
│  [온보딩] ──최초실행──→ [홈]                              │
│                         │                               │
│                    ┌────┴────┬────────┬────────┐       │
│                    ↓         ↓        ↓        ↓       │
│                  [홈]     [내역]   [통계]   [설정]       │
│                    │         │        │        │       │
│                    └────┬────┴────────┴────────┘       │
│                         │                               │
│                    [FAB 클릭]                           │
│                         ↓                               │
│                   [거래 입력 모달]                        │
│                         │                               │
│                    [저장/취소]                           │
│                         ↓                               │
│                   [이전 화면으로]                         │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

## 화면별 상세 설계

---

### 1. 온보딩 - 웰컴

**라우트:** `/onboarding`

#### 목적
- 앱의 정체성과 가치 전달
- 부담 없이 시작하도록 유도

#### 제공 경험
- "이건 가계부가 아니야, 거울이야" 인식
- 복잡하지 않겠구나 안심

#### 레이아웃

```
┌─────────────────────────────────────┐
│                                     │
│                                     │
│            [Logo]                   │  PinPig 로고 (SVG)
│                                     │
│            PinPig                   │
│                                     │
│      기록하는 가계부가 아니라         │
│         비춰주는 거울                │
│                                     │
│                                     │
│                                     │
│         [  시작하기  ]              │
│                                     │
│                                     │
└─────────────────────────────────────┘
```

#### 포인트
- 로고/마스코트 중앙 배치
- 태그라인으로 차별화 전달
- 버튼 하나로 단순하게
- 스킵 불가 (필수 통과)

#### 컴포넌트
- Logo
- Text (title, tagline)
- Button (primary)

### 2. 온보딩 - 예산 설정

**라우트:** `/onboarding/budget`

#### 목적
- 핵심 데이터(월 예산) 수집
- "남은 돈" 계산의 기준 설정

#### 제공 경험
- 빠르고 간편한 설정
- 나중에 바꿀 수 있다는 안심

#### 레이아웃

```
┌─────────────────────────────────────┐
│                                     │
│  ←                                  │
│                                     │
│     이번 달 쓸 수 있는 돈은          │
│         얼마인가요?                  │
│                                     │
│                                     │
│         1,500,000원                 │
│                                     │
│                                     │
│  ──────────────●──────────────      │
│  50만                       300만   │
│                                     │
│                                     │
│     나중에 설정에서 바꿀 수 있어요     │
│                                     │
│                                     │
│         [  완료  ]                  │
│                                     │
└─────────────────────────────────────┘
```

#### 포인트
- 질문형 문구로 자연스럽게
- 슬라이더로 빠른 입력
- 금액 직접 입력도 가능 (탭)
- "나중에 바꿀 수 있어요" 안심 문구

#### 컴포넌트
- Header (back button)
- Text (question)
- AmountDisplay
- Slider (50만~300만, 10만 단위)
- Text (helper)
- Button (primary)

### 3. 홈 (메인 대시보드)

**라우트:** `/`

#### 목적
- **1초 안에 "남은 돈" 확인**
- 오늘 써도 되는 돈 파악
- 최근 거래 빠른 확인

#### 제공 경험
- 앱 열자마자 핵심 정보
- 판단 없는 사실 전달
- "내 상태를 비춰주는" 느낌

#### 레이아웃

```
┌─────────────────────────────────────┐
│                                     │
│              1월                    │
│                                     │
│           847,000원                 │  ← Hero (가장 큼)
│      이번 달 쓸 수 있는 돈           │
│                                     │
│  ━━━━━━━━━━━━━━━━━━━━○━━━━━━━      │  ← 진행률
│                                     │
│     15일 남음 · 하루 56,000원       │  ← 일평균
│                                     │
├─────────────────────────────────────┤
│                                     │
│  최근 거래                          │
│                                     │
│  [Coffee] 스타벅스        오늘 14:32 │  Lucide: Coffee
│                           4,500원   │
│  ─────────────────────────────      │
│  [ShoppingBag] 쿠팡       오늘 11:20 │  Lucide: ShoppingBag
│                          32,000원   │
│  ─────────────────────────────      │
│  [Wallet] 급여                 어제  │  Lucide: Wallet
│                      + 3,000,000원  │
│                                     │
│            모두 보기 →              │
│                                     │
│            [ + ]                    │  FAB: Lucide Plus
├─────────────────────────────────────┤
│  [Home] [List] [BarChart2] [Settings]│  Bottom Nav (Lucide)

```

#### 정보 위계

1. **Hero** - 남은 돈 (가장 크고 눈에 띄게)
2. **Progress** - 예산 소진 상태
3. **Daily** - 하루 평균 쓸 수 있는 돈
4. **Recent** - 최근 거래 3건
5. **FAB** - 거래 추가

#### 포인트
- 남은 돈이 **Hero** (40px, Light)
- 진행률 바는 심플하게 (색상 변화 없음)
- 최근 거래는 3건만 (더보기로 이동)
- FAB은 항상 접근 가능

#### 컴포넌트
- MonthSelector
- BudgetHero
- ProgressBar
- DailyBudget
- RecentTransactions
- FAB
- BottomNav

### 7. 거래 입력 (모달)

**트리거:** FAB 클릭

#### 목적
- **3터치 이내 거래 기록**
- 빠르고 부담 없는 입력

#### 제공 경험
- 최소한의 입력으로 완료
- 자주 쓰는 카테고리 빠른 접근

#### 레이아웃

```
┌─────────────────────────────────────┐
│                                     │
│  [X]                        저장    │  Lucide: X
│                                     │
│     [ 지출 ]  [ 수입 ]              │  탭
│                                     │
│              45,000                 │  금액 (Hero)
│               원                    │
│                                     │
│  ─────────────────────────────      │
│                                     │
│  [Utensils] [Coffee] [Car] ...      │  카테고리 (Lucide 아이콘)
│    식비      카페    교통           │
│                                     │
│  ─────────────────────────────      │
│                                     │
│  [Calendar] 오늘 (1월 7일)      [>] │  Lucide: Calendar, ChevronRight
│  [FileText] 메모 추가           [>] │  Lucide: FileText, ChevronRight
│                                     │
├─────────────────────────────────────┤
│                                     │
│  ┌─────┬─────┬─────┐              │
│  │  1  │  2  │  3  │              │
│  ├─────┼─────┼─────┤              │
│  │  4  │  5  │  6  │              │  ← 숫자패드
│  ├─────┼─────┼─────┤              │
│  │  7  │  8  │  9  │              │
│  ├─────┼─────┼─────┤              │
│  │ 00  │  0  │  ⌫  │              │
│  └─────┴─────┴─────┘              │
│                                     │
└─────────────────────────────────────┘
```

#### 3터치 플로우

```
[FAB 탭] → [금액 입력 + 카테고리 선택] → [저장]
   1                  2                    3
```

#### 포인트
- 금액이 가장 크게 (Hero)
- 숫자패드 기본 표시
- 카테고리는 수평 스크롤 (자주 쓰는 순)
- 날짜 기본값 = 오늘
- 메모는 선택사항 (접힘)
- 저장 버튼 항상 활성화 (금액 0이면 비활성)

#### 스마트 기본값

```typescript
// 시간대별 카테고리 추천
const getDefaultCategory = () => {
  const hour = new Date().getHours();
  if (hour >= 7 && hour < 10) return 'transport';   // 출근
  if (hour >= 11 && hour < 14) return 'food';       // 점심
  if (hour >= 14 && hour < 17) return 'cafe';       // 오후
  if (hour >= 18 && hour < 21) return 'food';       // 저녁
  return 'etc';
};
```

#### 컴포넌트
- ModalHeader
- TypeToggle
- AmountDisplay
- CategoryPicker
- DatePicker
- MemoInput
- NumPad

## Bottom Navigation

### 구성

| 순서 | Lucide 아이콘 | 라벨 | 라우트 |
|------|--------------|------|--------|
| 1 | `Home` | 홈 | `/` |
| 2 | `List` | 내역 | `/history` |
| 3 | `BarChart2` | 통계 | `/stats` |
| 4 | `Settings` | 설정 | `/settings` |

### 스펙

```
높이: 56px + safe-area-bottom
```
