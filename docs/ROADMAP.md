## 5. 하지 않을 것들

핵심 철학에 반하거나 복잡도 대비 가치가 낮은 기능:

| 기능 | 제외 이유 |
|------|----------|
| 투자/자산 추적 | 복잡도 증가, 타겟 사용자 니즈 아님 |
| 타인과 비교 | 프라이버시, 스트레스 유발 |
| 게이미피케이션 | 검증되지 않음, 철학 불일치 |
| 영수증 OCR | 비용 대비 효용 불확실 |
| 복식부기 | 타겟 사용자 니즈 아님 |
| AI 지출 조언 | "판단 없는 비춤" 철학 위반 |
| 은행 연동 | 복잡도, 보안 이슈, Phase 1 범위 외 |

## 9. 결론

PinPig은 MVP를 성공적으로 완료했으며, 핵심 철학인 "비춰주는 거울"을 일관되게 구현하고 있습니다.

### 핵심 방향성

```
1. 단순함 유지
   - 기능 추가 시 복잡도 증가 경계
   - "하지 않을 것" 리스트 준수

2. 철학 일관성
   - 모든 기능은 "판단 없는 비춤" 원칙
   - 스트레스/죄책감 유발 요소 배제

3. 점진적 발전
   - PWA 완성 → iOS 전환
   - 사용자 피드백 기반 우선순위 조정
```
