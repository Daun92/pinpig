# MoneyMirror Design System

## 디자인 철학

### 핵심 정의

> **"기록하는 가계부가 아니라, 비춰주는 거울"**

머니미러는 금융앱이 아닌 **일상앱**이다. 
매일 쓰는 노트처럼, 돈도 일상의 기록일 뿐.
특별하지 않게. 부담 없이. 조용히.

---

### 첫인상 목표

사용자가 앱을 처음 봤을 때 느껴야 할 세 가지:

| 인상 | 달성 방법 |
|------|----------|
| **깔끔하다** | 요소 최소화, 정렬 철저, 일관된 여백 |
| **더할 게 없다** | 장식 제거, 기능에 필요한 것만 존재 |
| **편안하다** | 따뜻한 톤, 낮은 대비, 시각적 밀도 낮춤 |

---

### 디자인 원칙

#### 1. Quiet Interface
- 색상으로 소리치지 않는다
- 경고와 강조를 최소화한다
- 정보는 텍스트와 숫자로 전달한다

#### 2. Paper Metaphor
- 종이 위에 적는 느낌
- 따뜻한 오프화이트 배경
- 잉크처럼 부드러운 검정 텍스트

#### 3. Everyday Simplicity
- 금융 용어 대신 일상 용어
- 노트앱처럼 보이는 구조
- 거창하지 않은 일상 기록

#### 4. Breathe
- 충분한 여백은 기능이다
- 시각적 밀도를 낮춘다
- 한 화면에 하나의 주인공

---

### 포지셔닝

```
금융앱 느낌                    일상앱 느낌 (우리)
───────────────────────────────────────────────
파란색, 녹색                   모노톤, 따뜻한 중성색
차트, 그래프 강조               숫자와 텍스트 중심
"자산관리" "절약"              "기록" "확인"
경고, 알림 강조                 조용한 정보 전달
은행 앱 같은 구조               노트 앱 같은 구조
```

## 컬러 시스템

### Primary Palette: Warm Mono

모든 색상은 따뜻한 톤(Warm)을 기반으로 한다.
차가운 그레이(Blue-tinted)가 아닌 따뜻한 그레이(Yellow/Red-tinted).

#### 배경 (Paper)

| 이름 | HEX | 용도 |
|------|-----|------|
| Paper White | `#FAF9F7` | 메인 배경 |
| Paper Light | `#F5F4F1` | 카드, 입력창 배경 |
| Paper Mid | `#ECEAE6` | 구분선, 비활성 영역 |

#### 텍스트 (Ink)

| 이름 | HEX | 용도 |
|------|-----|------|
| Ink Black | `#1C1B1A` | 주요 숫자, 제목 |
| Ink Dark | `#3D3C3A` | 본문 텍스트 |
| Ink Mid | `#6B6966` | 보조 텍스트, 아이콘 |
| Ink Light | `#9C9A96` | 힌트, 타임스탬프 |

#### 기능색 (Semantic)

| 이름 | HEX | 용도 | 사용 원칙 |
|------|-----|------|----------|
| Positive | `#4A7C59` | 수입 표시 | + 기호와 함께만 |
| Caution | `#8B7355` | 주의 상태 | 텍스트로 대체 권장 |

**원칙:** 기능색은 최소한으로. 가능하면 기호(+, -)와 텍스트로 상태 전달.

## 타이포그래피

### 서체

| 용도 | 서체 | 이유 |
|------|------|------|
| 숫자 | SF Pro Display / Roboto | 시스템 폰트, 숫자 가독성 최적 |
| 한글/영문 | Pretendard | 깔끔함, 다양한 웨이트, 무료 |

### 웨이트 사용

- **Light (300):** 큰 숫자 (Hero)
- **Regular (400):** 본문, 일반 숫자
- **Medium (500):** 제목, 강조

Bold는 사용하지 않는다. 무겁지 않게.

### Type Scale

| 이름 | 크기 | 웨이트 | 용도 |
|------|------|--------|------|
| Hero | 40sp | Light | 메인 잔액 |
| Amount | 20sp | Regular | 거래 금액 |
| Title | 18sp | Medium | 섹션 제목 |
| Body | 15sp | Regular | 본문 |
| Sub | 13sp | Regular | 카테고리, 날짜 |
| Caption | 11sp | Regular | 타임스탬프 |

### 금액 표기 규칙

```
DO:
847,000원           원 뒤에, 쉼표 포함
+ 3,000,000         수입만 + 기호
4,500               지출은 기호 없이

DON'T:
₩847,000            통화 기호 앞에
-4,500              마이너스 기호 (스트레스 유발)
847000              쉼표 없이
```

## 아이콘 시스템

### 스타일 정의

| 속성 | 값 |
|------|-----|
| 타입 | Line Icon (외곽선) |
| 두께 | 1.5px |

| 기본 크기 | 20px |

단순화된 형태, 텍스트 없이도 인식 가능해야 함.

| 카테고리 | 형태 설명 |
|----------|----------|
| 식비 | 포크+나이프 (십자 형태) |
| 카페 | 컵 (단순 사각형 + 손잡이) |
| 교통 | 바퀴 두 개 |
| 쇼핑 | 쇼핑백 |
| 주거 | 지붕 형태 |
| 의료 | 십자 |
| 여가 | 플레이 버튼 |
| 통신 | 말풍선 |
| 금융 | 동전 (원) |
| 기타 | 점 세 개 |

### UI 아이콘

|------|------|
| 추가 | + |

## 컴포넌트

### 카드

```
배경: Paper Light (#F5F4F1) 또는 투명
테두리: 없음 또는 1px Paper Mid
그림자: 없음
모서리: 12px
패딩: 20px
```

그림자 없음 = 평면적 = 종이 위에 올린 느낌

### 구분선

```
색상: Paper Mid (#ECEAE6)
두께: 1px
마진: 좌우 콘텐츠와 정렬
```

### 버튼

#### Primary
```
배경: Ink Black (#1C1B1A)
텍스트: Paper White (#FAF9F7)
패딩: 14px 24px
모서리: 8px
```

#### Secondary
```
배경: 투명
텍스트: Ink Black (#1C1B1A)
테두리: 1px Paper Mid (#ECEAE6)
```

#### Ghost (Text Button)
```
배경: 투명
텍스트: Ink Mid (#6B6966)
```

### 진행률 바

```
배경: Paper Mid (#ECEAE6), 높이 2px
진행: Ink Black (#1C1B1A), 높이 2px
```

마커, 색상 변화 없음. 상태는 텍스트로 전달.

### 리스트 아이템

```
구조: [아이콘 20px] [간격 12px] [텍스트] [금액]
높이: 자동 (패딩 기반)
구분: 1px 라인
```

## 모션

### 원칙

**최소한, 부드럽게.**

있어야 할 이유가 없으면 넣지 않는다.

### 허용되는 모션

| 상황 | 값 |
|------|-----|
| 화면 전환 | 200ms fade |
| 요소 등장 | 150ms fade-in |
| 숫자 변경 | 100ms ease-out |
| 리스트 등장 | stagger 30ms |

### 금지되는 모션

- 바운스
- 스프링
- 과장된 효과
- 화려한 카운트업
- 파티클

## 음성과 톤

### 언어 원칙

| DO | DON'T |
|-----|-------|
| 쓸 수 있는 돈 | 잔여 예산 |
| 오늘 | 금일 |
| 기록하기 | 지출 등록 |
| 56,000원 남음 | 예산 잔액 56,000원 |

### 피드백 메시지

중립적 관찰. 칭찬도 비난도 아님.

```
DO:
"예산의 80%를 사용했어요"
"하루 56,000원 남았어요"
"이번 달 식비가 늘었어요"

DON'T:
"예산을 초과했습니다!"
"절약을 더 하셔야 해요"
"지출이 너무 많아요"
```

## 체크리스트

디자인 리뷰 시 확인:

### 첫인상
- [ ] 화면에 불필요한 요소가 있는가? → 있으면 제거
- [ ] 여백이 충분한가?
- [ ] 색상이 3개를 넘는가? → 넘으면 줄이기

### 일상앱 느낌
- [ ] 금융 용어 대신 일상 용어 사용했는가?
- [ ] 경고/강조가 과하지 않은가?
- [ ] 노트앱처럼 보이는가?

### 가시성
- [ ] 아이콘으로 카테고리 구분 가능한가?
- [ ] 숫자가 잘 읽히는가?
